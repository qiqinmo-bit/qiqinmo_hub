# 📖 飞书开放平台配置教程

> 让飞书「灵感收集」Bot 能接收你的消息并自动入库

---

## 总览

配置完成后，你在飞书给 Bot 发消息：

```
你: Cursor 的 Composer 功能可以同时编辑多个文件...
↓
飞书服务器 → POST → localtunnel 隧道
  → feishu_bot.py (本机:5676)
    → process_inspiration.py (本机:5677)
      → 01_原始灵感/ → git push → AI 分析 → 知识文档
```

---

## 第一步：打开飞书开放平台

> 浏览器打开：**[https://open.feishu.cn/app](https://open.feishu.cn/app)**

登录你的飞书账号。

---

## 第二步：找到你的应用

在「我的应用」列表中找到 **「灵感收集」**，点击进入。

> 如果没有看到该应用，点击右上角「创建应用」→ 输入名称「灵感收集」→ 创建

---

## 第三步：配置事件订阅（最关键）

左侧菜单 → **事件订阅**

### 3.1 设置回调地址

在「回调地址」输入框粘贴以下地址：

```
https://wet-goats-switch.loca.lt/feishu/webhook
```

点击 **「添加」** 按钮。

> 如果提示"地址校验失败"，不要慌——这是因为飞书服务器还没完成验证，点击「完成」继续即可。

### 3.2 添加事件

在「事件」区域，点击 **「添加事件」**：

1. 在搜索框输入 `im.message.receive_v1`
2. 勾选搜索结果中的 `im.message.receive_v1`
3. 点击 **「确认添加」**

---

## 第四步：配置权限

左侧菜单 → **权限管理**

点击 **「添加权限」**：

1. 在搜索框输入 `im:message`
2. 勾选 `im:message`（描述：读取用户发给机器人的消息）
3. 点击 **「确认添加」**

> 如果还有其他权限提示（比如 `im:resource`），也一并勾选。

---

## 第五步：添加 Bot 能力

左侧菜单 → **应用功能 → 机器人**

确保 **「机器人」** 开关已开启（绿色）。

> 如果没看到这个菜单，说明你的应用类型不对。回到「凭证与基础信息」，确认应用类型是「企业自建应用」。

---

## 第六步：发布上线

右上角 **「发布」** 按钮 → **「创建版本」**：

| 字段 | 填写 |
|------|------|
| 版本号 | `1.0.0` |
| 更新说明 | `灵感收集 Bot 首次上线，自动将消息转为知识文档` |

点击 **「保存」** → 在弹出的确认框点击 **「上线」**

> ⚠️ 如果你的飞书管理员有审批流程，需要等待审批通过。

---

## 第七步：找到你的 Bot

在飞书 App 中：

1. 点击搜索框
2. 搜索 **「灵感收集」**
3. 点击进入 Bot 聊天界面
4. 发送任意消息（如：`测试一下`）

---

## 第八步：验证是否成功

### 本地检查

在终端查看 feishu_bot.py 的运行日志，应该能看到：

```
[Feishu] 收到消息: ...
[Feishu] 处理消息: 测试一下...
  ✅ 灵感工作流已保存
```

### 文件检查

查看 `01_原始灵感/` 目录下是否有新的文件夹生成：

```bash
dir 01_原始灵感\
```

会看到类似 `2026-06-09_测试一下/` 的文件夹。

### 完整链路验证

```bash
python _scripts/process_inspiration.py status
```

应该能看到总条目数增加了。

## 常见问题

### Q: 回调地址填了但提示验证失败

A: 无视警告，点击「完成」继续。因为 localtunnel 的域名飞书可能需要几分钟才能访问到。如果持续失败：

1. 确认 feishu_bot.py 在运行 → `curl http://localhost:5676/feishu/health`
2. 确认隧道在运行 → `curl https://wet-goats-switch.loca.lt/feishu/health`
3. 确认防火墙没拦截 5676 端口

### Q: 飞书搜索不到「灵感收集」Bot

A: 还没发布成功。回到第六步，确认「发布」→「创建版本」→「上线」三步都完成了。

如果是审批流程卡住了，去飞书工作台找「审批」应用处理。

### Q: Bot 回复了「灵感已收录」但没看到文件

A: 检查 webhook 服务（端口 5677）是否在运行：

```bash
curl http://localhost:5677/webhook/inspiration -X POST -H "Content-Type: application/json" -d "{\"test\":true}"
```

### Q: localtunnel 地址变了

A: localtunnel 每次重启都会生成新地址。你需要在飞书开放平台更新回调地址。

如果当前隧道断开，重新启动：

```bash
node _scripts/start_tunnel.mjs
```

然后用新地址更新飞书开放平台的回调配置。
